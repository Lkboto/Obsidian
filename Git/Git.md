# 📘 Introducción a Git

## 📌 ¿Qué es Git?

Git es un sistema de control de versiones distribuido diseñado para llevar el seguimiento de cambios en proyectos de software (aunque puede usarse para cualquier tipo de archivo).

Fue creado por Linus Torvalds en 2005 y actualmente es el sistema de control de versiones más utilizado en el mundo.

Con Git podemos:

- Controlar cambios en nuestro código  
- Volver a versiones anteriores  
- Trabajar en equipo sin sobrescribir el trabajo de otros  
- Gestionar diferentes versiones de un proyecto mediante ramas  

---

## ⚙️ ¿Por qué usar Git?

- ✔ Control total sobre el historial del proyecto  
- ✔ Trabajo colaborativo eficiente  
- ✔ Integración con plataformas como GitHub  
- ✔ Seguridad y respaldo del código  

---

## 🧠 Conceptos básicos

Antes de profundizar, es importante entender algunos conceptos fundamentales:

- **[[Repositorios]]** → Proyecto gestionado por Git  
- **[[Commits]]** → Confirmación de cambios  
- **Branch (rama)** → Línea paralela de desarrollo  
- **Merge** → Unión de ramas  
- **Remote** → Repositorio remoto (por ejemplo, en GitHub)
